WORKFLOW

四步，从意图到构建配置。

不是凭印象开写代码，而是先把「要做什么」和「依据什么」固定在一份文件里。

①

刷新来源并探索

读取 Agent SDK 全部文档、全部 demo，并在改进模式审查现有项目。

②

WHAT WE CAN DO

带逐页、逐项目链接展示当前能做的完整能力。

③

HAVE / SUGGEST

说明已有能力与缺口，再提出最小、可追溯的改进建议。

④

最终确认

用户确认 create / modify / keep / verify 计划后才开始写文件。

EXPORT

一份 JSON，
交给下一棒。

创建模式把需求交给 new-agent-sdk；改进模式复用同一套文档与验证规则，只做目标所需的最小 diff。两种模式都支持聊天内交互和 HTML 表单。

✓

构建需求完整

项目意图、能力清单、补充要求，一个字段不少。

✓

引用可逐条核验

官方文档逐 URL、示例逐项目，并附选择理由。

✓

一键复制为提示词
